# Release v0.4.13

**Contract-parity focus for the `devlead`, `devrev`, and `qa` agentic roles.**

## Overview

This release sharpens three agentic roles — `devlead`, `devrev`, and `qa` — around **contract parity**: validating that public API shape, defaults, enums, and error paths match the synced SSOT schemas, standards, and fixtures, not merely that happy-path tests pass. The role prompts gain explicit parity checklists, mindset principles, and guardrails, and are bumped to role version 1.0.1.

The role-prompt updates are content-only (no schema, public-API, or standards changes) and propagate to all four language wrappers. This release also carries a **tooling/process-hardening pass** that brings Crucible onto the goneat v0.5.12 conventions used across the Fulmen galaxy, so assets synced into downstream libraries arrive already-formatted.

## Highlights

### Contract-parity role updates (role version 1.0.0 → 1.0.1)

**`devlead`** — implementation gains an explicit contract-parity discipline:

- New mindset principles: implement strict/contract-compliant behavior first, then add tolerant modes explicitly; verify schema + fixtures + standards before declaring complete; default precedence (schema-silent → standards authoritative; schema-vs-standards conflict → escalate before merge).
- New responsibilities: validate public API shape and defaults against synced schemas/standards; add fixture-backed parity tests for happy-path and failure-path behavior; escalate intentional SSOT deviations before merge.
- New `implementation` checklist covering contract, defaults, error-path, deferred-scope, fixture, and cross-language parity, plus quality gates.
- Scope refinement: code review and PR oversight move to `devrev`/`qa`; `qa` added to the "distinct from" context; quality-gate guidance updated `make precommit` → `make check-all`.

**`devrev`** — review now asserts contract conformance:

- Verify contract parity for public APIs (types, fields, enums, defaults) against synced schemas, fixtures, and standards.
- New schema/fixture parity review checklist; guardrail against approving on green CI alone when parity checks are missing.

**`qa`** — acceptance now requires parity validation:

- Schema + standard parity and fixture parity (happy-path + failure fixtures) added to the acceptance checklist; guardrail against green-CI-only sign-off.

### Language wrapper propagation

All updates are synced to the `lang/python`, `lang/rust`, and `lang/typescript` role catalogs (`config/agentic/roles/`). The Rust navigation catalog (`lang/rust/src/agentic.rs`) is regenerated from the updated YAML.

## Tooling & Process Hardening

Crucible is the oldest infoarch repo in the galaxy and the original proving ground for goneat's parallel tree-walking formatter. This release aligns it with the current galaxy tooling baseline:

- **goneat pin `v0.5.3` → `v0.5.12`** (Makefile). The repository YAML was reformatted to v0.5.12 conventions so the SSOT assets synced into `tsfulmen`, `pyfulmen`, and `rsfulmen` arrive already-formatted — eliminating format churn at the consumer.
- **Root `.yamllint` and `.yamlfmt`** (2-space convention) added, matching `tsfulmen`/`goneat`, to stabilize `actionlint`/`yamllint` behavior and end recurring CI spacing disagreements.
- **Guardian git-hook browser-approval gate removed** from `.goneat/hooks/{pre-commit,pre-push}`. Crucible moves to the standard commit/push/PR flow (repo/merge-policy controls), retiring the local browser-interception model that suited the earlier microteam direct-commit era.
- **Makefile: `lint-check` and `pr-final` targets.** `lint-check` is a non-mutating goneat format+lint check for CI prep. `pr-final` is the PR-flow successor to `prepush`: it runs the non-mutating check, the full prep, then a drift guard that fails if prep left uncommitted tracked-file changes.

No public schema, API, or standards content changed in this pass — it is formatting and developer-workflow only.

## Dependency Refresh

A conservative minor/patch dependency wave (no majors — TypeScript 6, vitest 4, pytest 9, and the glob/ejs/thiserror majors are deferred to a future coordination epic):

- **biome** 2.4.2 → 2.4.16 (root + lang/typescript), with `biome.json` `$schema` aligned to the installed CLI (2.4.16). (goneat v0.5.12 bundles biome 2.4.6, so `goneat assess` may still flag the schema until its bundled biome catches up; the repo's biome and CI are unaffected.)
- **js-yaml** 4.1.1 → 4.2.0; **@types/bun** → 1.3.14; **@types/node** held on the v20 line (20.19.41).
- **pytest-cov** → 7.1.0; **ruff** 0.14.0 → 0.15.15. The newer ruff flagged UP042 on the `fulencode`/`fulpack` enums, modernized `(str, Enum)` → `StrEnum` (Python floor is 3.12); the test suite confirms unchanged behavior.
- **CI** `setup-bun` `bun-version` 1.2.1 → 1.3.9.

(Rust crate deps float within unchanged `Cargo.toml` caps — `lang/rust/Cargo.lock` is not committed, so there is no lockfile change in this release.)

## Breaking Changes

**None.** Role-prompt content only — no schema, public-API, standards, or slug changes. Consumers that load these roles see updated prompt text and new optional checklist fields. The tooling/formatting changes do not alter any asset's parsed content.
