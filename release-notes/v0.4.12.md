# Release v0.4.12

**Typed role catalog API across all four language implementations, three new roles, and full schema field coverage.**

## Overview

This release ships the typed role catalog API for Go, TypeScript, Python, and Rust тАФ the first cross-language API for loading, parsing, and enumerating agentic role definitions from the embedded catalog. All four implementations return a fully-typed `RolePrompt` with complete schema coverage. Three new roles are added (`cxotech`, `deliverylead`, `infraeng`), the upstream sync advances to 3leaps/crucible v0.1.12, and two cross-cutting schema compliance bugs identified during library team reviews are fixed before tagging.

## Highlights

### Typed Role Catalog API тАФ All Four Languages

Consumers can now load typed role definitions without manual YAML parsing:

**Go**:

```go
import "github.com/fulmenhq/crucible"

role, err := crucible.LoadRole("devlead")
fmt.Println(role.Name, role.Responsibilities)

catalog, err := crucible.LoadRoleCatalog()   // map[string]*RolePrompt

slugs, err := crucible.ListRoleSlugs()       // sorted []string
```

**TypeScript**:

```typescript
import { loadRole, loadRoleCatalog, listRoleSlugs } from "@fulmenhq/crucible";

const role = loadRole("devlead"); // RolePrompt | undefined
const catalog = loadRoleCatalog(); // Map<string, RolePrompt>
const slugs = listRoleSlugs(); // string[]
```

**Python**:

```python
from crucible.agentic import load_role, load_role_catalog, list_role_slugs

role = load_role("devlead")                  # RolePrompt
catalog = load_role_catalog()                # dict[str, RolePrompt]
slugs = list_role_slugs()                    # list[str]
```

**Rust** (codegen тАФ navigation catalog):

```rust
use crucible_codegen::agentic::Role;

let role = Role::from_slug("devlead").unwrap();
let name = role.metadata().name;             // &'static str
for r in Role::all() { println!("{}", r); }
```

### Full Schema Field Coverage

All three runtime implementations (Go, TypeScript, Python) now expose every field
defined in `role-prompt.schema.json`. Three fields were previously missing:

| Field                | Type                                   | Used By         |
| -------------------- | -------------------------------------- | --------------- |
| `pre_push_checklist` | string[]                               | releng, devlead |
| `required_reading`   | `{ description?, pattern?, files?[] }` | releng          |
| `cross_role_note`    | string                                 | releng, secrev  |

The `required_reading` field is typed as a structured object (`RoleRequiredReading`)
matching the shape used in the YAML files тАФ not a flat string list.

### Slug Regex Aligned to Schema

All implementations now validate slugs against the canonical schema pattern
`^[a-z][a-z0-9]*$` (lowercase letters and digits, must start with a letter).
Prior implementations used a more permissive regex that allowed hyphens and
underscores not permitted by the schema. No existing role slugs are affected.

### Rust Codegen тАФ Navigation Catalog

The Rust `Role` enum and `RoleMetadata` struct in `lang/rust/src/agentic.rs` are
generated at build time from the YAML catalog. `RoleMetadata` is intentionally a
**navigation catalog** тАФ it carries the fields needed for role dispatch and display
(slug, name, description, status, scope, responsibilities) with zero heap allocation
(`&'static` strings). For full `RolePrompt` data, rsfulmen implements runtime YAML
loading via its own `src/crucible/roles.rs` following the foundry catalog pattern.

## Breaking Changes

**None.** All new functions and types are additive. Existing APIs (`GetConfig`, `ConfigRegistry`, schema and standards registries) are unchanged. The slug regex tightening is technically a validation change but no existing role slug is affected.

## Migration Guide

**Library teams integrating the typed role API:**

See `release-notes/` in each library repo's upgrade memo for team-specific integration steps.

**Slug validation**: If you have custom code that validates role slugs, align to
`^[a-z][a-z0-9]*$`. No existing slugs use hyphens or underscores.

**RolePrompt consumers**: If you were accessing `pre_push_checklist`, `required_reading`,
or `cross_role_note` via raw YAML (e.g., `GetConfig("agentic/roles/releng.yaml")`), you
can now use the typed API and get these fields directly.
