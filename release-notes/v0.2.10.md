**Date**: 2025-11-15

## Highlights

- **Extension Framework Foundation**: Module registry infrastructure with platform-modules/foundry-catalogs split, taxonomy-based schemas, and comprehensive validation tooling
- **Canonical FaÃ§ade Principle**: Documented pattern ensuring cross-language consistency and unified error envelopes across all helper library modules

## What's New

### 1. Extension Framework Foundation

Infrastructure enabling helper library module extensibility with clear governance, validation, and cross-language consistency patterns.

#### Module Registry Split

Separation of module registry into two specialized registries:

**Platform Modules** (`config/taxonomy/library/platform-modules/v1.0.0/modules.yaml`):

- **Purpose**: Code modules providing functionality (algorithms, utilities, operations)
- **Count**: 17 modules (7 Core, 10 Common, 0 Specialized)
- **Examples**: logging, signals, pathfinder, config-path, fulpack
- **Implementation Transparency**: Documents stdlib vs third-party dependencies per language

**Foundry Catalogs** (`config/taxonomy/library/foundry-catalogs/v1.0.0/catalogs.yaml`):

- **Purpose**: Reference data catalogs (lookup tables, taxonomy data, static resources)
- **Count**: 7 catalogs
- **Examples**: exit-codes, country-codes, http-status-groups, mime-types, signals
- **Characteristics**: Read-only, language-agnostic, versioned separately from code

**Rationale**: Distinct update cadence (catalogs change rarely), different testing requirements (data validation vs behavior tests), clearer mental model for consumers.

#### Module Entry Schema

Taxonomy-based schema for module registration (`schemas/taxonomy/library/modules/v1.0.0/module-entry.schema.json`):

**Three-Tier System**:

- **Core Tier** (7 modules): Always present in helper libraries, foundational functionality
  - Examples: logging, signals, error-handling, crucible-shim
- **Common Tier** (10 modules): Default installation, frequently needed
  - Examples: pathfinder, config-path, three-layer-config, fulpack
- **Specialized Tier** (0 modules): Opt-in extensions for specialized use cases
  - Future: fulencoding (character encoding detection/conversion)

**Implementation Transparency Field**:

```yaml
go:
  status: available
  implementation: stdlib # or "third-party", "custom"
  dependencies: ["archive/tar", "compress/gzip"]
```

**Evidence Pointers**:

- Schemas: JSON schemas defining module data structures
- Config: YAML/JSON configuration catalogs
- Fixtures: Test data, examples, reference implementations

**Cross-Language Status Tracking**:

- `planned`: Not yet implemented
- `available`: Implemented and tested
- `deprecated`: Scheduled for removal

#### Validation Script

Comprehensive module registry validator (`scripts/validate-module-registry.ts`) with 7 automated checks:

1. **Orphan Detection**: Identify schemas/config/fixtures without registry entries
2. **Dead Entry Detection**: Identify registry entries without artifacts
3. **Evidence Validation**: Verify schema/config/fixture files exist at declared paths
4. **Cross-Language Status**: Ensure each module has status for go/python/typescript
5. **Schema Compliance**: Validate registry YAML against module-entry.schema.json
6. **Tier Rules**: Enforce tier-specific constraints (Core always available, etc.)
7. **Cross-Reference Validation**: Verify module dependencies exist

**Artifact Discovery**: Automatically scans `schemas/`, `config/`, `fixtures/` directories to detect unregistered modules.

**Target**: Zero errors, zero warnings validation (achieved).

**Integration**: Added to `make validate-schemas` and `make check-all` targets.

#### Canonical FaÃ§ade Principle

New section in Helper Library Standard documenting fundamental design pattern:

**Core Principle**: Helper libraries provide canonical faÃ§ades for ALL functionality, including stdlib wrappers.

**Why FaÃ§ade Everything**:

- **Cross-Language Consistency**: Same API surface across Go/Python/TypeScript
- **Unified Error Envelopes**: All operations return FulmenError with code/message/context
- **Taxonomy-Driven Design**: Module registry defines complete API surface
- **Implementation Flexibility**: Language teams choose stdlib vs third-party per ecosystem norms

**Single API Surface Principle**:

```go
// âœ… CORRECT - Import from helper library
import "github.com/fulmenhq/gofulmen/fulpack"
archive, err := fulpack.Create(source, output, "tar.gz", options)

// âŒ INCORRECT - Direct stdlib import
import "archive/tar"
tw := tar.NewWriter(output)  // No error envelope, no taxonomy consistency
```

**Implementation Strategy Matrix**:

| Module Type    | Go Strategy                     | Python Strategy       | TypeScript Strategy      |
| -------------- | ------------------------------- | --------------------- | ------------------------ |
| **Archive**    | stdlib (archive/tar)            | stdlib (tarfile)      | third-party (archiver)   |
| **Encoding**   | third-party (golang.org/x/text) | third-party (chardet) | third-party (iconv-lite) |
| **Pathfinder** | stdlib (filepath, os)           | stdlib (pathlib)      | stdlib (path, fs)        |

**When to Wrap Stdlib**:

- âœ… Operation appears in module registry (platform-modules.yaml)
- âœ… Cross-language consistency required (API parity)
- âœ… Error handling needs unification (FulmenError envelope)
- âœ… Taxonomy integration needed (foundry catalogs, exit codes)
- âŒ Pure language-specific utility with no cross-language equivalent

### 2. Fulpack Archive Module Specification

Complete Common-tier archive module specification with Pathfinder integration, security-by-default protections, and cross-language deterministic behavior requirements. Specification only in v0.2.10 - implementation validation follows in helper libraries.

#### Taxonomy Schemas (3 files)

**Archive Formats** (`schemas/taxonomy/library/fulpack/archive-formats/v1.0.0/formats.yaml`):

```yaml
formats:
  tar.gz:
    key: tar_gz
    display_name: "Tarball (gzipped)"
    file_extensions: [".tar.gz", ".tgz"]
    mime_types: ["application/gzip", "application/x-gzip"]
    stdlib_support:
      go: true # archive/tar + compress/gzip
      python: true # tarfile
      typescript: false # third-party: archiver
    features:
      compression: true
      streaming: true
      preserves_permissions: true
      preserves_symlinks: true
```

**Operations** (`schemas/taxonomy/library/fulpack/operations/v1.0.0/operations.yaml`):

- `create`: Create archives from files/directories
- `extract`: Extract with explicit destination (no CWD)
- `scan`: List entries without extraction (Pathfinder integration)
- `verify`: Validate integrity, checksums, detect threats
- `info`: Get metadata without extraction (format detection, size estimation)

**Entry Types** (`schemas/taxonomy/library/fulpack/entry-types/v1.0.0/types.yaml`):

- `file`: Regular file
- `directory`: Directory entry
- `symlink`: Symbolic link (platform-dependent support)

#### Data Structure Schemas (8 files)

All schemas in `schemas/library/fulpack/v1.0.0/`:

1. **archive-info.schema.json**: Archive metadata
   - Format, total size, compressed size, compression ratio
   - Entry counts, checksum algorithm

2. **archive-entry.schema.json**: Entry metadata
   - Path, type, size, checksum, permissions mode
   - Timestamps (created, modified)

3. **archive-manifest.schema.json**: Complete table of contents
   - Array of entries with optional indexes (by path, by type)
   - Total/compressed sizes

4. **validation-result.schema.json**: Integrity verification results
   - Valid flag, checksum matches, issues detected
   - Threat analysis (path traversal, decompression bombs)

5. **create-options.schema.json**: Archive creation parameters
   - Compression level, checksum algorithm
   - Include/exclude patterns, symlink handling

6. **extract-options.schema.json**: Extraction parameters
   - Explicit destination (required), overwrite policy
   - Security limits (max size, max entries, max compression ratio)

7. **scan-options.schema.json**: Scanning parameters
   - Pattern matching for Pathfinder integration
   - Include/exclude filters

8. **extract-result.schema.json**: Extraction operation results
   - Extracted count, total bytes, elapsed time
   - Skipped entries, errors encountered

#### Canonical API (5 Operations)

Verb-based operations working across all formats:

**create(source, output, format, options?)**:

```typescript
// TypeScript example
const archive = await fulpack.create(
  "./src", // source directory
  "./dist/src.tar.gz", // output archive
  "tar.gz", // format
  {
    compression: 6, // gzip level 1-9
    checksum: "sha256",
    exclude: ["*.log", "node_modules"],
  },
);
```

**extract(archive, destination, options?)**:

```go
// Go example - explicit destination required (no CWD)
result, err := fulpack.Extract(
  "./dist/src.tar.gz",  // archive path
  "./output",           // explicit destination
  &fulpack.ExtractOptions{
    Overwrite: false,  // default: error on conflict
    MaxSize: 100 * 1024 * 1024,  // 100MB limit
    MaxEntries: 10000,
  }
)
```

**scan(archive, options?)**:

```python
# Python example - Pathfinder integration
entries = fulpack.scan(
  "./dist/src.tar.gz",
  options={
    "pattern": "**/*.go",  # glob pattern
    "include_directories": False
  }
)
# Returns: List[ArchiveEntry] for Pathfinder matching
```

**verify(archive, options?)**:

```go
// Go example - integrity validation
result, err := fulpack.Verify(
  "./dist/src.tar.gz",
  &fulpack.VerifyOptions{
    ChecksumAlgorithm: "sha256",
    DetectThreats: true,  // path traversal, bombs
  }
)
```

**info(archive)**:

```typescript
// TypeScript example - metadata without extraction
const info = await fulpack.info("./dist/src.tar.gz");
console.log(info.format); // "tar.gz"
console.log(info.totalSize); // 1024000
console.log(info.compressedSize); // 204800
console.log(info.compressionRatio); // 5.0
```

#### Security Model (Mandatory Protections)

All implementations MUST enforce security protections:

**1. Path Traversal Protection**:

- âŒ Reject entries with `../` path components
- âŒ Reject absolute paths (`/etc/passwd`, `C:\Windows\System32`)
- âœ… Only allow relative paths within destination directory

**2. Decompression Bomb Protection**:

- Monitor compression ratio (reject >100x by default)
- Enforce max uncompressed size (configurable, 100MB default)
- Enforce max entry count (configurable, 10000 default)

**3. Checksum Verification**:

- SHA-256 default (configurable: md5, sha1, sha256, sha512)
- Fail extraction on checksum mismatch
- Log checksum validation results

**4. Safe Defaults**:

- Explicit destination directory required (no CWD extraction)
- Error on overwrite (require explicit `overwrite: true`)
- No symlink following by default
- Preserve permissions on Unix (configurable)

**Implementation Testing**: `pathological.tar.gz` fixture contains security test cases (path traversal attempts, absolute paths, symlink escapes).

#### Pathfinder Integration

Unified glob searches across filesystems and archives:

**Pattern**: `scan()` operation returns `ArchiveEntry[]` compatible with Pathfinder APIs

```typescript
// Search across filesystem AND archive
const fsResults = await pathfinder.find("**/*.go", { basePath: "./src" });
const archiveResults = await fulpack.scan("./dist/src.tar.gz", {
  pattern: "**/*.go",
});

// Both return compatible Entry structures for unified processing
```

**Performance Target**: Archive TOC scan <1s (no extraction)

**Edge Cases Documented**:

- Symlinks (platform-dependent handling)
- Invalid UTF-8 in filenames (reject or transcode)
- Absolute paths (reject with ABSOLUTE_PATH error)
- Path traversal attempts (reject with PATH_TRAVERSAL error)

**Cross-Language Determinism**: Same glob pattern MUST return same entries across Go/Python/TypeScript implementations.
